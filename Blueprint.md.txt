Spending Leak Detector
Local-First Deterministic Personal Finance Analytics Platform
0. Mission

Build a local-first, deterministic personal finance analytics platform named:

Spending Leak Detector

This is NOT a banking app.
This is NOT an AI budgeting assistant.

It is a privacy-preserving analytics engine with a lightweight web UI that ingests CommBank CSV exports and helps users:

understand spending month-by-month

detect leaks and lifestyle creep

separate financial movement from consumption

auto-categorize merchants deterministically

track savings goals with feasibility math

receive explainable encouragement and corrective suggestions

ML is optional future phase only.

Everything must be deterministic.

1. Core Principles (Non-Negotiable)

Runs locally via Docker Compose

Same input → same output

User always controls categorization

Movement ≠ Spending

No online APIs or merchant lookups

Monthly analytics is primary time grain

Backend + data modeling first, UI thin

Privacy first (no account numbers stored)

6. Merchant Normalization (Deterministic)

Algorithm:

Uppercase

Remove punctuation

Strip:

CARD XX####

VALUE DATE

city/state suffixes

Remove prefixes:

PAYPAL *

SQ *

VISA PURCHASE

DIRECT DEBIT

Collapse whitespace

Result becomes merchant_key.

7. Movement Detection

Set is_movement = true if description contains:

TRANSFER TO
TRANSFER FROM

9. Definitions

Expense:
amount < 0 AND is_movement = false

Income:
amount > 0 AND is_movement = false

Movement:
is_movement = true

10. Review Queue Rules

Include:

category == Unknown

is_movement AND abs(amount)>=500

abs(amount)>=100 AND discretionary

Discretionary:
Eating Out, Food Delivery, Shopping, Entertainment, Subscriptions, Other, Unknown

11. Monthly Analytics

month = DATE_TRUNC('month', txn_date)

All insights operate monthly.

12. Core Formulas

income_total = sum(amount>0 AND not movement)

expense_total = sum(abs(amount<0 AND not movement))

net = income - expense

savings_rate = net / income

required_monthly =
(goal_amount - reported_total_savings) / months_remaining

historical_saving =
median(net_last_6_months)

feasibility_ratio =
historical / required

Thresholds:

≥1.2 Comfortable
0.9–1.2 On track
0.7–0.9 At risk
<0.7 Unrealistic

Category spike:
current > median(last6) * 1.4

Lifestyle creep:
avg(last3) - avg(prev3)

Spend velocity:
first_7_days / total_month

Opportunity cost:
extra * 12 * 0.06
